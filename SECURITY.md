# Security policy

AI Workspace is not yet released and has no supported versions.

Please do not report suspected vulnerabilities in a public issue. Use GitHub's
private vulnerability reporting feature for this repository when available,
or contact the repository owner privately through their published GitHub
profile.

Include a clear description, affected component, reproduction steps, impact,
and any suggested mitigation. Do not include real credentials, customer data,
private transcripts, or reversible anonymization mappings in the report.

Security-sensitive areas include secret handling, anonymization, model routing,
prompt injection, tool permissions, artifact storage, plugin trust, and local
workspace isolation.
